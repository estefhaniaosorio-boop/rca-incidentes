Resumen Ejecutivo

Se analizaron 87 incidentes del tablero MDSB. El análisis agrupa los fallos en 12 soluciones maestras que cubren 5 sistemas: SAI (45 incidentes), Análisis Digital (15), SIOS (12), Facturación Electrónica (5) y Huella (3). Un bloque significativo (17 tickets) corresponde a problemas de acceso/infraestructura (Java, VPN, certificados). Las 3 soluciones de mayor impacto cubrirían el 55% de los incidentes.

SOL-01: Pagos, Siniestros y Estado de CuentaAlta · 4 sprints

1.1 Validación pre-pago (nuevo middleware)

FUNCTION validar_elegibilidad_pago(p_solicitud_id, p_siniestro_id) RETURN BOOLEAN
BEGIN
  -- 1. Verificar solicitud NO retirada
  SELECT estado INTO v_estado FROM solicitudes WHERE id = p_solicitud_id;
  IF v_estado IN ('RETIRADA', 'TERMINADA') THEN RETURN FALSE; END IF;

  -- 2. Verificar siniestro NO terminado
  SELECT estado INTO v_est_sin FROM siniestros WHERE id = p_siniestro_id;
  IF v_est_sin = 'TERMINADO' THEN RETURN FALSE; END IF;

  -- 3. Verificar fecha de desocupacion
  SELECT fecha_desocupacion INTO v_fecha FROM solicitudes WHERE id = p_solicitud_id;
  IF v_fecha IS NOT NULL AND v_fecha < SYSDATE THEN RETURN FALSE; END IF;

  -- 4. Limite de indemnizacion REAL
  SELECT COUNT(DISTINCT periodo) INTO v_meses_pagados
  FROM pagos_realizados
  WHERE siniestro_id = p_siniestro_id AND estado = 'PAGADO';

  SELECT limite_meses INTO v_limite FROM parametros_indemnizacion
  WHERE solicitud_id = p_solicitud_id;

  IF v_meses_pagados >= v_limite THEN RETURN FALSE; END IF;

  -- 5. Anti-duplicado
  SELECT COUNT(*) INTO v_dup FROM pagos_realizados
  WHERE siniestro_id = p_siniestro_id AND periodo = p_periodo AND estado = 'PAGADO';
  IF v_dup > 0 THEN RETURN FALSE; END IF;

  RETURN TRUE;
END;

1.2 Constraint anti-duplicados

ALTER TABLE pagos_realizados
ADD CONSTRAINT uk_pago_periodo_siniestro UNIQUE (siniestro_id, periodo, tipo_amparo);

1.3 Corregir límite de indemnización

-- ANTES (incorrecto):
SELECT meses_cumplidos FROM limite_indemnizacion WHERE solicitud_id = :id;

-- DESPUES (correcto):
SELECT COUNT(DISTINCT TO_CHAR(fecha_pago, 'YYYY-MM'))
FROM pagos_realizados
WHERE siniestro_id = :siniestro_id AND estado = 'PAGADO'
  AND tipo NOT IN ('REVERSION', 'ANULADO');

1.4 Saneamientos días 30-31

-- ANTES: WHERE TO_CHAR(fecha_terminacion, 'DD') <= '28'
-- DESPUES:
WHERE fecha_terminacion BETWEEN TRUNC(SYSDATE, 'MM') AND LAST_DAY(SYSDATE)

Criterios de Aceptación

AC1: Dado siniestro terminado por pago directo, cuando batch ejecuta, entonces NO programa pago.
AC2: Dado solicitud retirada, cuando se ejecuta facturación, entonces NO genera giro.
AC3: Dado siniestro con 24 meses pagados y límite 36, entonces muestra 24/36.
AC4: Dado intento de giro duplicado, entonces BD rechaza con constraint violation.
AC5: Dado solicitudes terminadas días 30-31, entonces aparecen en listado 3-2-35.

2.2 Health-check post-despliegue

#!/bin/bash
ENDPOINTS=("https://saiprod.bolnet.com.co:443/health"
           "https://gestion.ellibertador.co/dashboard-ed/api/health"
           "https://www.ellibertador.co/indemnizaciones-web/health")
for url in "${ENDPOINTS[@]}"; do
  status=$(curl -s -o /dev/null -w "%{http_code}" "$url" --max-time 10)
  [ "$status" != "200" ] && echo "FALLO: $url = $status" && alertar
done
AC1: Dado despliegue en producción, cuando finaliza, entonces health-check valida todos los endpoints en <2 min.
AC2: Dado certificado próximo a expirar (7 días), entonces se renueva automáticamente.

function onDocumentUploaded(solicitudId, documentId) {
  const sol = getSolicitud(solicitudId);
  if (['PENDIENTE','CON_RESULTADO','PROCESADA'].includes(sol.estado)) {
    sol.estado = 'PENDIENTE_REASIGNACION';

      fechaIngreso: new Date(), documentoId: documentId

}

3.2 Job de reconciliación (cada 15 min)

// Cron: */15 * * * *
const huerfanas = await db.query(`
  SELECT s.id FROM solicitudes s
  JOIN documentos d ON d.solicitud_id = s.id
  LEFT JOIN cola_asignacion c ON c.solicitud_id = s.id AND c.estado = 'PENDIENTE'
  WHERE d.fecha_carga > s.fecha_ultimo_resultado
    AND c.id IS NULL AND s.estado NOT IN ('CERRADA','ANULADA')
`);
for (const sol of huerfanas) {
  await colaAsignacion.push({ solicitudId: sol.id, prioridad: 'RECONCILIACION' });
}
if (huerfanas.length > 5) await alertar(`${huerfanas.length} huerfanas`);

SOL-06: Sincronización de ParámetrosMedia · 2 sprints
CREATE TABLE parametros_negocio (
  id NUMBER PRIMARY KEY, clave VARCHAR2(100) UNIQUE,
  valor VARCHAR2(500), fecha_vigencia DATE,
  sistema_origen VARCHAR2(50), fecha_update TIMESTAMP DEFAULT SYSTIMESTAMP
);

-- Job diario de reconciliacion:
SELECT p.clave, p.valor AS central, sai.valor AS sai_val, sios.valor AS sios_val
FROM parametros_negocio p
LEFT JOIN sai_parametros sai ON sai.clave = p.clave
LEFT JOIN sios_parametros sios ON sios.clave = p.clave
WHERE p.valor != NVL(sai.valor,'NULL') OR p.valor != NVL(sios.valor,'NULL');
AC1: Dado cambio de parámetro central, entonces SAI y SIOS reflejan nuevo valor en <24h.
AC2: Dado discrepancia, entonces genera alerta al equipo.

08. Reversión de Terminación

PROCEDURE revertir_terminacion(p_id NUMBER) IS BEGIN
  UPDATE solicitudes SET estado='ACTIVA', fecha_terminacion=NULL WHERE id=p_id;
  DELETE FROM descuentos WHERE solicitud_id=p_id AND tipo='TD_DEV'
    AND fecha_creacion > (SELECT fecha_term FROM audit_sol WHERE solicitud_id=p_id);
  recalcular_estado_cuenta(p_id);
  UPDATE siniestros SET estado='ACTIVO' WHERE solicitud_id=p_id AND estado='SUSPENDIDO';
  COMMIT;
END;
